# Support

## Getting Help

Use these channels based on your request type:

- Bug reports: open an issue using the bug template.
- Feature requests: open an issue using the feature request template.
- Usage questions: open an issue and clearly label it as a question.
- Security concerns: email izeno.contact@gmaill.com or follow `SECURITY.md`.

## Before Opening an Issue

1. Read `README.md` and relevant docs in `docs/`.
2. Search existing issues and pull requests.
3. Test against the latest `main` branch when possible.
4. Include minimal reproduction steps.

## Helpful Details to Include

- Izel source snippets that reproduce the issue.
- Exact command(s) run.
- Expected behavior and actual behavior.
- Platform details (OS, Rust toolchain, LLVM version).
- Logs, diagnostics, or stack traces.

## Maintainer Availability

Support is provided on a best-effort basis. Response time can vary depending on maintainer availability and issue complexity.
